# 程序员口袋 - 添加新工具指引

本文档详细说明如何向"程序员口袋"项目添加一个新工具。

## 概览

添加一个新工具需要：

1. **修改 2 个现有文件**：配置中心 + 应用配置
2. **创建 4 个新文件**：工具页面的 JS、JSON、WXML、WXSS
3. **可选：添加图标**：如果工具使用新的图标

## 步骤详解

假设要添加一个名为"JSON格式化"的工具，ID 为 `jsonformat`。

---

### 步骤 1：修改 `config/tools.js`

**文件位置**：`/config/tools.js`

在数组中添加工具元数据：

```javascript
// 工具配置中心
module.exports = [
  { id: 'imgcompress', name: '图片压缩', icon: 'image', page: '/pages/tools/imgcompress/index' },
  // 新增下面这行：
  { id: 'jsonformat', name: 'JSON格式化', icon: 'code', page: '/pages/tools/jsonformat/index' }
]
```

**字段说明**：

- `id`: 工具的唯一标识符（必须小写，不包含空格）
- `name`: 工具的显示名称（在工具列表中显示）
- `icon`: 图标名称（需要在 `components/icon/index.js` 中定义）
- `page`: 工具页面的完整路径

---

### 步骤 2：修改 `app.json`

**文件位置**：`/app.json`

在 `pages` 数组中添加新页面路径：

```json
{
  "pages": [
    "pages/tools/index",
    "pages/tools/imgcompress/index",
    "pages/tools/jsonformat/index",  // 新增这行
    "pages/recent/index",
    "pages/profile/index"
  ]
  // ... 其他配置
}
```

---

### 步骤 3：创建工具页面目录

**创建目录**：`/pages/tools/jsonformat/`

该目录下需要 4 个文件：

#### 3.1 创建 `index.js` - 页面逻辑

**文件路径**：`/pages/tools/jsonformat/index.js`

```js
Page({
  data: {
    input: '',
    output: '',
    error: ''
  },  onLoad() {
    console.log('jsonformat tool loaded')
  },
})
```


#### 3.2 创建 `index.json` - 页面配置

**文件路径**：`/pages/tools/jsonformat/index.json`

```json
{
  "navigationBarTitleText": "JSON格式化"
}
```

#### 3.3 创建 `index.wxml` - 页面模板

**文件路径**：`/pages/tools/jsonformat/index.wxml`

```wxml
<view class="page">
  <view class="container">
  
    <!-- 页面标题 -->
    <view class="title">JSON格式化</view>
    <view class="subtitle">快速格式化和校验JSON</view>

  </view>
</view>
```

#### 3.4 创建 `index.wxss` - 页面样式

**文件路径**：`/pages/tools/jsonformat/index.wxss`

---

### 步骤 4：（可选）添加新图标

如果工具使用了新的图标（如上例中的 `code` 已存在），无需添加。

如果需要新增图标，修改 `components/icon/index.js` 中的 `renderSvg()` 方法：

**文件位置**：`/components/icon/index.js`

在 switch 语句中添加新的 case：

```javascript
renderSvg(name, color, size){
  const stroke = color
  const sw = 1.8
  switch(name){
    case 'clock': return `...`
    // ... 其他已有的图标
  
    // 新增自定义图标
    case 'myicon': return `<svg xmlns='http://www.w3.org/2000/svg' viewBox='0 0 24 24'>
      <path d='M12 2L15 8H9Z' fill='${stroke}'/>
      <circle cx='12' cy='14' r='4' stroke='${stroke}' stroke-width='${sw}' fill='none'/>
    </svg>`
  
    default: return `...`
  }
}
```

**图标设计建议**：

- 使用 24x24 的 viewBox
- 优先用 `stroke`（描边）而不是 `fill`（填充）
- `stroke-width` 统一使用 `${sw}`（约 1.8）
- 颜色使用 `${stroke}` 变量以支持动态配置

## 完整示例

假设添加一个"二维码生成"工具：

### 1. 修改 config/tools.js

```javascript
{ id: 'qrcode', name: '二维码生成', icon: 'qrcode', page: '/pages/tools/qrcode/index' }
```

### 2. 修改 app.json

```json
"pages/tools/qrcode/index"
```

### 3. 创建 pages/tools/qrcode/ 目录

按照上述步骤 3 创建 4 个文件。

### 4. 添加图标（如果需要）

在 `components/icon/index.js` 中添加：

case 'qrcode': return `<svg xmlns='http://www.w3.org/2000/svg' viewBox='0 0 24 24'>

## 常见问题

**Q: 工具列表中没有看到新工具？**
A: 检查 `config/tools.js` 和 `app.json` 是否都修改了。重新编译小程序。

**Q: 工具页面无法打开？**
A: 确认 `app.json` 中的 `pages` 数组包含了正确的路径。

**Q: 图标显示不出来？**
A: 检查 `config/tools.js` 中的 `icon` 值是否在 `components/icon/index.js` 中有对应的 case。

**Q: 工具页面样式不对？**
A: 使用项目提供的 glass-card 组件和统一的样式类名，参考现有工具页（如 imgcompress）。
